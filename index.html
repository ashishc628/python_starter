01Chapter One
The Origin Story

What is Python?

A language designed to be read by humans, accidentally named after a British comedy troupe, and now quietly running about half of modern software.

Python is a high-level, general-purpose programming language. Translation: you tell a computer what you want, in something close to English, and Python figures out how to make the machine do it. It was created by a Dutch programmer named Guido van Rossum in 1991, who took the name not from the snake but from Monty Python's Flying Circus. (You'll find spam and eggs hiding in the documentation to this day.)

Today, Python powers the recommendation engine on Netflix, the artificial intelligence behind ChatGPT, scientific simulations at NASA, financial models at Wall Street firms, and probably the homework of every CS student in the country. Its superpower is simple: it gets out of your way.

"Python is executable pseudocode. Perl is executable line noise." — an old programmer's joke that, unfortunately for Perl, has aged well.

Things you can build

Why Python, of all things?

  • Readable. The syntax looks like a recipe, not algebra.
  • Batteries included. A massive standard library means you rarely start from zero.
  • Versatile. Web apps, data science, AI, automation, game prototypes, scripts — all in one language.
  • Beginner-friendly, expert-deep. You can write your first program in an hour and still be learning new things ten years in.

03Chapter Three
Storing Things

Variables & Types

A variable is just a name with a value attached. Picture a labelled jar on a shelf: you put something inside, and you can fetch it back later by the label.

In Python you don't have to declare what kind of thing a variable will hold. You just write name = value and Python figures it out.

variables.py
Python 3

Each variable above holds a different type of value. Python recognises four basic ones to start with:

int
Whole numbers, positive or negative
age = 17
float
Numbers with a decimal point
pi = 3.14
str
Text, wrapped in quotes
name = "Ada"
bool
True or False — nothing else
on = True

04Chapter Four
Doing Things

Operators & Math

Once you have values, you'll want to combine them. Python uses the same symbols as a calculator, plus a few extras worth knowing.

The standard arithmetic operators are +, -, *, /. Python adds ** for powers, % for remainders, and // for integer division. Tweak the numbers and run it:

math.py
Python 3

Comparison & logic

Comparisons return True or False. They're the building blocks of decisions, which we'll meet in Chapter 7.

Quick reference

  • == equal to  ·  != not equal
  • < less than  ·  > greater than
  • <= at most  ·  >= at least
  • and, or, not — combine boolean values
Watch out: = assigns a value, while == tests for equality. A single equals sign is a verb; a double equals sign is a question.
05Chapter Five
Working with Text

Strings, and what to do with them

Text is the raw material of conversation. Python gives you a small toolkit for cutting, joining, and re-shaping it — and one elegant trick called the f-string.

Strings can be added together (called concatenation), repeated, indexed by position, and sliced. Best of all, they can be slotted into a sentence with f-strings:

Strings also come with useful built-in methods. Try chaining a few:

reference

name.upper()       # "  ADA LOVELACE  " — all caps

06Chapter Six
Collections

Lists

Most data in the world is plural. A grocery list. The students in a classroom. Frames in an animation. Python's list is your everyday container.

A list is written with square brackets, items separated by commas. Lists can hold anything — numbers, strings, even other lists.

lists.py
Python 3
Python is zero-indexed: the first element is at position 0, not 1. This trips up everyone exactly once. It's a rite of passage.
07Chapter Seven
Making Decisions

If this, then that

A program that always does the same thing isn't terribly useful. The if statement gives your code the power to take different paths depending on what's true.

Python uses indentation (the whitespace at the start of a line) to decide what belongs inside the if. This is one of Python's most distinctive features — the layout is the structure.

grading.py
Python 3

The shape of an if

  • if — the first thing to check
  • elif — short for "else if", checked only if everything above is false
  • else — the catch-all when nothing matched
  • Every line inside a branch must be indented (4 spaces is the convention)

08Chapter Eight
Repetition

Loops

Computers are tireless. Where humans get bored at the second repetition, a computer happily does the same thing a billion times in a row. Loops are how you ask them to.

Python has two kinds of loops. A for loop walks through a sequence one item at a time. A while loop keeps going as long as some condition holds.

loops.py
Python 3
A common rookie error: a while loop whose condition never becomes false. Your program runs forever. (Don't worry, the browser will eventually stop it. Probably.)
09Chapter Nine
Reusable Recipes

Functions

When you find yourself copy-pasting the same lines of code, it's time to bottle them up and give them a name. That bottle is called a function.

Define a function with def. Give it a name, list its inputs (called parameters), and write what it should do. Use return to hand a value back to the caller.

functions.py
Python 3

Why bother with functions?

  • Less repetition — write it once, call it everywhere.
  • Easier to read — a well-named function tells you what, not how.
  • Easier to fix — one change updates every caller.
  • Easier to test — each piece can be checked in isolation.

def celsius_to_fahrenheit(c):

print(celsius_to_fahrenheit(100))  # 212.0
print(celsius_to_fahrenheit(37))   # 98.6 — body temperature

Part B — Your take-home challenge

Open a Python file on your computer (or use replit.com if you don't have Python installed yet), and turn the one-shot game above into a real interactive loop. Here's the spec — you write the code.

The Brief

  1. Pick a random secret number between 1 and 100.
  2. Give the player 6 attempts to guess it.
  3. On each guess, tell them higher or lower.
  4. If they guess right, celebrate. If they run out, reveal the answer.
  5. Show how many attempts they used.

Stretch goals

  • Add a difficulty selector (easy = 50, hard = 100).
  • Tell them "freezing / cold / warm / hot" based on distance.
  • Wrap the whole thing in a function called play().
  • Ask "play again?" at the end and loop.
  • Keep a running scoreboard of best attempts.
